# 日志分析提示词

## 用途

用于让 Agent 根据日志、异常栈、接口请求信息、traceId 或测试失败输出进行问题定位。

## 输入要求

执行前应尽量获取：

- 错误日志原文
- 异常时间
- 接口路径或任务名称
- traceId / requestId
- 请求参数与响应
- 相关环境
- 复现步骤

缺少信息时应标注“待确认”，不要编造调用链或根因。

## 分析维度

- 问题现象
- 关键日志
- 异常类型
- 影响范围
- 初步判断
- 需要补充的证据
- 建议验证方式
- 建议修复方向

## 推荐输出结构

1. 问题现象
2. 关键日志摘录
3. 初步原因判断
4. 证据链
5. 待确认信息
6. 建议排查步骤
7. 建议修复方向

## 禁止事项

- 不要把猜测写成确定根因。
- 不要省略关键日志证据。
- 不要只说“可能是后端问题”。
- 不要编造不存在的 traceId、SQL、接口或异常栈。
