# 038｜真实任务验证包

## 目录用途

本目录用于沉淀一组可直接复制给 AI 助手执行的真实测试任务样例，用来验证当前个人 AI 测试助手是否真的能覆盖日常测试工作。

本验证包不是测试平台，不是团队协作平台，也不引入新的复杂架构。当前目标是用真实任务反向验证已有能力是否有用，避免继续堆功能。

## 验证范围

1. 需求分析
2. 测试用例生成
3. 日志分析
4. pytest + Allure 报告总结
5. Codex 修改结果审查
6. 接口测试设计
7. 失败测试结果生成缺陷报告
8. 成熟工具接入评估

## 文件说明

| 文件 | 用途 |
| --- | --- |
| `001_requirement_analysis.md` | 验证助手能否对需求进行规则拆解、风险识别和测试范围分析 |
| `002_test_case_generation.md` | 验证助手能否按固定表格格式生成可复制的中文测试用例 |
| `003_log_analysis.md` | 验证助手能否从日志中定位失败环节、风险和后续排查方向 |
| `004_pytest_allure_summary.md` | 验证助手能否基于 pytest + Allure 结果生成客观测试摘要 |
| `005_codex_change_review.md` | 验证助手能否审查 Codex 修改结果是否符合项目边界 |
| `006_api_test_design.md` | 验证助手能否设计接口测试方案、数据库校验点和一致性校验点 |
| `007_failed_test_to_bug_report.md` | 验证助手能否把失败测试结果整理成专业缺陷报告 |
| `008_tool_integration_evaluation.md` | 验证助手能否评估成熟工具是否适合当前项目接入 |

## 使用方式

1. 打开任意一个 `examples/real_tasks/*.md` 文件；
2. 复制其中的任务文本；
3. 粘贴到 ChatGPT / Codex / OpenHands / PyCharm AI / 命令行入口；
4. 观察 AI 输出是否符合预期；
5. 根据输出效果决定是否沉淀为 prompt、skill、workflow 或 template。

## 评价标准

每个任务至少从以下维度判断：

1. 是否能理解任务目标；
2. 是否能输出结构化结果；
3. 是否能贴近测试工作实际；
4. 是否能避免空泛建议；
5. 是否能给出可执行下一步；
6. 是否符合个人 AI 测试助手定位；
7. 是否避免重新造轮子；
8. 是否避免引入平台化、团队化、业务写死等偏离方向。

## 当前边界

本目录只保存任务样例，不要求自动执行，不要求接入真实业务系统，不生成自动化代码，不继续拆 Allure，不开放 `shell` / `filesystem_write` / `github_write` 能力。

后续建议进入 `039：真实任务跑测与评分表`，用统一评分表逐个验证这些任务的输出质量。
