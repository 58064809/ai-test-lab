# 002｜测试用例生成任务验证

## 任务

请根据以下需求生成测试用例。

## 需求

用户登录支持手机号验证码登录、账号密码登录、第三方授权登录。

登录成功后返回 `token`、用户信息、是否新用户标识。

如果是新用户，需要引导完善资料。

连续输错密码 5 次后锁定账号 30 分钟。

验证码 5 分钟内有效，同一手机号 60 秒内只能发送一次验证码。

## 输出要求

1. 使用 Markdown 表格；
2. 列固定为：`标题｜优先级｜前置条件｜测试步骤｜预期结果`；
3. 优先级使用 `P0`、`P1`、`P2`、`P3`；
4. 不需要“用例类型”列；
5. 不需要一个步骤对应一个预期，支持多对多；
6. 覆盖正常、异常、边界、安全、兼容性场景；
7. 输出中文；
8. 表格内容要整齐、专业、可直接复制到飞书或 Excel。
