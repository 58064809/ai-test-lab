# 缺陷报告提示词

## 用途

用于让 Agent 根据测试现象、复现步骤、日志、截图、接口请求或数据库证据整理缺陷报告。

## 输入要求

执行前应尽量获取：

- 问题标题或现象
- 测试环境
- 复现步骤
- 实际结果
- 预期结果
- 影响范围
- 日志、截图、请求响应或数据证据

缺少信息时标注“待补充”，不要编造证据。

## 推荐输出结构

1. 缺陷标题
2. 测试环境
3. 前置条件
4. 复现步骤
5. 实际结果
6. 预期结果
7. 影响范围
8. 严重程度建议
9. 附件 / 日志 / 证据
10. 初步定位建议

## 禁止事项

- 不要把没有证据的猜测写成根因。
- 不要编造日志、接口返回或数据库数据。
- 不要省略复现步骤。
- 不要只写一句“功能异常”。
