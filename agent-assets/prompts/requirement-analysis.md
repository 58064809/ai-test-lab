# 需求分析提示词

## 用途

用于让 Agent 对需求、PRD、原型、接口变更或业务说明进行测试视角分析。

## 输入要求

执行前应尽量获取需求原文、原型说明、接口文档、字段说明、业务规则、用户角色、状态流转、影响范围和上线范围。

缺失内容不得自行补全为确定结论，应标注“待确认”。

## 分析维度

- 需求目标
- 功能范围
- 主流程
- 分支流程
- 角色与权限
- 状态流转
- 数据一致性
- 接口影响
- 前端影响
- 后端影响
- 数据库 / Redis / MQ 影响
- 风险点
- 待确认问题
- 测试重点

## 推荐输出结构

1. 需求目标
2. 功能范围
3. 核心流程
4. 角色与权限
5. 状态与数据流转
6. 接口与数据影响
7. 测试重点
8. 风险点
9. 待确认问题
10. 建议测试范围

## 禁止事项

- 不要编造需求中没有的规则。
- 不要把猜测写成确定事实。
- 不要默认套用当前公司的电商业务规则。
- 不要只输出泛泛而谈的测试建议。
