# API 测试工作流

## 用途

用于指导 Agent 在接口测试任务中按证据和契约工作，避免凭空生成无法执行的接口测试。

## 前置输入

执行前应尽量收集：

- OpenAPI / Swagger / Apifox / Postman / 接口说明
- 测试环境 base URL
- 鉴权方式
- 请求参数与响应结构
- 业务规则
- 高风险接口清单

缺少信息时标注“待确认”。

## 工作步骤

1. 读取接口契约或接口说明。
2. 确认鉴权、环境、请求方法、路径、参数、响应结构。
3. 识别正向、反向、边界、权限、幂等、异常、数据一致性场景。
4. 如存在 OpenAPI，优先评估 Schemathesis。
5. 如可基于真实调用生成测试，评估 Keploy。
6. 生成测试用例或自动化脚本。
7. 明确验证命令和风险边界。

## 禁止事项

- 不要编造接口路径、参数、状态码或响应字段。
- 不要直接对生产环境执行破坏性测试。
- 不要对支付、发券、扣库存、发短信等高风险接口自动压测或 fuzz。
