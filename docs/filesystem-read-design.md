# filesystem_read 接入前设计

## 目标

- 为未来 `filesystem_read` 能力定义最小安全边界
- 限制未来只读文件访问只落在仓库内低风险工程文件
- 在真实接入 MCP 前，先固化白名单、黑名单、路径穿越拦截和 dry-run 语义

## 非目标

- 不真实读取任何文件内容
- 不真实接入 MCP Server
- 不调用 MCP SDK
- 不开放 `filesystem_write`
- 不读取真实业务敏感文件

## 候选成熟方案

- 基于成熟 filesystem MCP Server 的只读能力
- 基于未来 MCP 工具适配层的仓库内只读文件访问
- 不考虑自研文件协议或自研执行器

## 选择策略

- 第一阶段只设计“路径是否允许被读取”的安全策略
- 正式接入时优先复用成熟 MCP 文件系统工具
- 任何真实读取前，都必须先经过路径白名单和敏感模式拦截

## 权限等级

- 对应 `docs/mcp-security-policy.md` 中的 `L2: read_project_files`
- 当前仍然只允许 dry-run 级评估，不允许真实读取

## 路径白名单

第一阶段仅允许未来读取以下仓库内路径：

- `README.md`
- `AGENTS.md`
- `docs/`
- `agent-assets/`
- `configs/*.yaml`
- `src/ai_test_assistant/`
- `tests/`
- `validation/`

## 敏感文件黑名单

第一阶段必须禁止未来读取以下内容：

- `.env`
- `.assistant/`
- `.git/`
- `*.pem`
- `*.key`
- `*.crt`
- `*token*`
- `*secret*`
- `*password*`
- 数据库连接配置
- 任何真实业务导出数据

## 读取限制

- 只接受仓库相对路径
- 必须阻止路径穿越，例如 `../.env`、`docs/../../.env`
- 不允许绝对路径
- 不允许访问黑名单目录和敏感模式
- 即使路径位于白名单目录下，只要命中敏感模式也必须拒绝

## dry-run 行为

- 当前只返回结构化策略判断
- 不读取文件内容
- 不调用任何 MCP Server
- 不调用本地命令

## 真实接入前置条件

- 目标 MCP 文件系统工具支持 Windows
- 能限制为仓库内只读访问
- 能显式关闭写入能力
- 能与当前策略模型对齐
- 能在接入前完成敏感路径和路径穿越验证

## Windows 注意事项

- 需要统一处理 `\` 和 `/`
- 需要阻止盘符绝对路径
- 需要避免大小写差异绕过黑名单

## 后续替换策略

- 当前策略模型只是安全边界占位，不是执行器
- 未来真实接入时，优先让 MCP adapter 调用这层策略
- 如果成熟 MCP 工具的路径限制能力更强，可以用它替代部分本地策略判断
