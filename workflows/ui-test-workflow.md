# UI 自动化工作流

## 用途

用于指导 Agent 在 UI 自动化任务中优先选择稳定、高价值、可维护的场景。

## 前置输入

执行前应尽量收集：

- 页面入口
- 用户角色
- 核心流程
- 测试环境
- 测试账号
- 原型或页面说明
- 历史高频回归问题

缺少信息时标注“待确认”。

## 工作步骤

1. 确认业务主链路和高风险页面。
2. 优先选择冒烟、核心回归、历史高频问题场景。
3. 判断是否需要 Playwright MCP 辅助浏览器操作。
4. 设计稳定选择器策略。
5. 生成可维护的 UI 测试方案或脚本。
6. 明确截图、trace、失败日志和报告产物。

## 禁止事项

- 不要追求全量 UI 自动化覆盖。
- 不要依赖脆弱 XPath。
- 不要编造页面按钮、字段和交互。
- 不要在未确认环境和账号时执行高风险业务操作。
