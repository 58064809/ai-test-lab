# 缺陷分析工作流

## 用途

用于指导 Agent 根据测试现象、日志、接口请求、截图、数据库证据等信息进行缺陷分析和报告整理。

## 前置输入

执行前应尽量收集：

- 问题现象
- 测试环境
- 复现步骤
- 实际结果
- 预期结果
- 日志或异常栈
- 请求参数与响应
- 截图或录屏
- 相关数据证据

缺少信息时标注“待补充”。

## 工作步骤

1. 还原问题现象。
2. 整理可执行复现步骤。
3. 提取关键日志、请求响应和数据证据。
4. 区分前端、后端、数据、依赖、环境、配置等可能方向。
5. 标注证据充分程度。
6. 输出缺陷报告。
7. 给出后续验证建议。

## 禁止事项

- 不要把没有证据的猜测写成确定根因。
- 不要编造日志、SQL、接口响应或截图内容。
- 不要省略复现步骤。
- 不要用“可能有问题”代替证据链。
